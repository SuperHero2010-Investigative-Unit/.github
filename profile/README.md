# SuperHero2010 Investigative Unit — Rules

This organization is an independent, one-person-led investigative body.
Its purpose is to **collect, preserve, and present evidence** of deceptive practices by other companies *(Most common is AI coding companies)*.

## 1. Organization Structure

- **Administrator:** SuperHero2010.
  + All final decisions are made by the administrator.
  + No one else has the authority to merge PRs, create new repositories, or change organization settings.

- **Lead investigator:** [SuperHero2010 (Detective)](https://youtube.com/@SuperHero2010-Detective) (fictional).
  + He is the one who presents the evidence on TV.
  + After the administrator has gathered and arranged all the necessary evidences, he will be the one to create the video to upload to his YouTube channel.

- **Authorized Members (if any):**
  + Can help review PRs, triage issues, or manage evidence.
  + **Must ask the administrator for approval** before taking any action that affects the organization.
  + Acting without approval will result in immediate removal of authorization.

- **Contributors (Public):**
  + Anyone can submit Pull Requests or open Issues.
  + PRs will be reviewed by the administrator (or an authorized member with administrator approval).
  + Only the administrator can merge PRs.

## 2. Repository Rules

- **Primary Repository:** The organization has **exactly one primary repository** for archiving evidence and commit history.

- **New Repositories:**
  + Authorized members can request to create a new repository **only for evidence-related purposes** (e.g., investigating a new company).
  + **You must ask the administrator first.**
  + Creating a repository without permission is a violation of these rules.

- **No Application Development:**
  + **You are NOT allowed to create repositories for application development** (apps, tools, engines, or any software product).
  + This organization exists for **investigation and evidence**, not for building products.
  + Violation of this rule will result in **immediate dismissal** from the organization.

## 3. Contributing Evidence

- Anyone can submit a Pull Request with new evidence (CSV files, chat logs, screenshots, screen recordings, emails, etc.).

- **PR Requirements:**
  + Evidence must be **authentic and unaltered**.
  + Include a clear description of what the evidence shows.
  + If possible, include steps to reproduce the issue.

- **Review Process:**
  + The administrator (or an authorized member with administrator approval) will review the PR.
  + The administrator can request changes or clarifications.
  + Only the administrator can merge the PR.

## 4. Prohibited Uses of Evidence

- **DO NOT use any evidence from this organization to threaten, intimidate, harass, or extort individuals or companies.** This organization exists to **expose the truth**, not to attack or harm.
- **Violation of this rule** will result in **immediate dismissal** from the organization. Depending on the severity, the administrator can also take legal action or report the violation to relevant authorities.

## 5. Handling Fake or Fabricated Evidence

- **If someone submits fake or fabricated evidence:**
  + The PR will be **immediately rejected** and **marked as fraudulent**.
  + The submitter will be **warned** (first offense).
  + A second offense will result in **permanent ban** from contributing.
  + The administrator can also publicly note that the submitter has a history of submitting fake evidence (without disclosing personal information).

- **If a member is found to have intentionally fabricated evidence:**
  + They will be **immediately dismissed** from the organization.
  + All their contributions will be **reviewed and potentially removed**.
  + The administrator can issue a public correction if fake evidence was previously accepted.

## 6. Conflict Resolution

- The **administrator is the final decision-maker** for all disputes.
- If a conflict arises between members, the administrator will investigate and issue a binding ruling.
- If a member disagrees with a ruling, they can appeal in writing. The administrator will review and issue a final decision.

## 7. Code of Conduct

- Be respectful.
- Focus on **evidence and facts**, not personal attacks.
- Do not harass or threaten individuals or companies.
- Do not share false or misleading information.
- Violations can result in removal from the organization.

## 8. Dismissal Policy

- Any member who violates the rules (especially **Rule 2: No Application Development** or **Rule 4: Prohibited Uses of Evidence**) will be **immediately dismissed** from the organization.
- Dismissed members lose all access to organization repositories and settings.

## 9. Amendments

- These rules can be amended by the administrator at any time.  
- Members will be notified of significant changes.
